

一个设计游戏概念的平台。基于以下几个原则：

1. 巧妇难为无米之炊，为了激发用户对游戏设计的灵感，我们的平台以**素材概念**为核心。
2. 文本存在，则逻辑上存在。素材的核心是它的名称和描述，因为现在的AI可以直接据此大量生图。
3. 设计与实现完全解耦，本平台仅专注于游戏设计，完全不涉及游戏实现。



先把“素材“定义好（一阶段开发）

- 素材的核心是它的名称和描述。
- 素材可以是一个小物件、一个场景、甚至是一个角色。
- 素材总是服务于游戏，是游戏美术的基本组成部分。

- 素材可以有配图，但这只是可选的附加项，因为配的图是可以直接根据文本描述，利用生成式AI生图服务直接得到的。
- 素材存在于公开素材库，可以被用户直接fork一份到自己的素材库。（会保留fork的记录，也就是记录这条素材由哪一个素材fork而来）
- 公开素材库的素材是不能被修改的，只能由用户公开，或是由发布者删除。被删除的公开素材不能再被fork，而且无法看到它的名称、描述、配图、发布者，但可以看到它fork自哪一个公开素材（若有）。
- 素材的创造可以由用户直接手动输入，只需要提供名称与描述。用户可以为素材写注释，但这个信息不会被生图AI使用。
- 素材的创造可以是用户与AI聊天的产物。用户可以创建一个与AI的对话，这个对话专门服务于素材生成，AI可以根据用户意图生成素材名称与描述（例如，用户想要一个赛博朋克风格的垃圾桶），并且根据素材本身的复杂度决定是否需要细化描述。
  - 用户与AI的聊天是可以有主题的，在这一段对话中用户可能围绕这个主题产生很多素材的灵感，这些素材（名称与描述）都可以暂存在这段会话中，并且随时可以导出到个人素材库。
  - 聊天服务于用户素材的产生，但它本身也可以给以灵感。因此它可以被归档，移除工作区，存在某个地方。

当“名称”与“描述”确定好之后，用户可以继续完善这个素材：（二阶段开发）

核心功能是素材图像的生成

- 素材已经有了完整描述，那么当前的AI就可以完全根据描述生图。
- 用户在素材详情界面可以为其生图，生图时可以额外添加描述，但这些额外描述也需要被记录。于是便生好了一张图。
- 一个素材概念可以有多张图，它们可以是基于素材本身的描述，额外添加不同的描述，也可以是相同描述多次生图。
- 素材可以选择一个图作为展示，但没有图的素材也必须能够正常展示，因为图像只是附加，核心仍然是素材的名称与描述。
- 对于公开的素材，用户是无法在它上面直接生图的，但用户完全可以fork一份到自己的私有库，然后自己生图。
- 不同素材是独立的（除了fork树），图片是素材的附属，二者地位并不平等。



当图像也具备了之后（也可以没有），用户可以创建游戏项目，把概念具象化：（三阶段开发）

核心功能是项目的创建，也就是真正的游戏设计

+ 游戏项目是私人的，暂时不要做公开。

+ 游戏项目可以从私人库里导入素材或素材包，他们可以有图也可以没有。
+ 游戏的设计可以是用户与AI共同探讨，AI可见用户的素材（文本）。
+ 提供情绪板，或者canvas，供用户将素材拼贴上去，甚至可以模拟玩法（视觉上）。
