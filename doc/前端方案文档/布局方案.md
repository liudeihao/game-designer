布局设计方案需要覆盖整个平台的空间逻辑，从最顶层的全局框架，一直到每个核心页面的具体分区。

------

### 全局布局框架

整个平台使用**三层嵌套空间模型**：Shell（壳）→ Workspace（工作区）→ Content（内容区）。

**Shell** 是永远不变的最外层，包含左侧的垂直导航栏（48px 宽，主区只有图标，无文字标签）。这条导航栏是全局的唯一锚点，无论用户在哪个页面，它始终在场。主区承载五个图标：探索（公开库）、我的库、会话、项目、用户。栏底固定为账户区：偏好设置（显示与字体）、圆形用户头像（占位：取显示名或用户名首字符，后续可接真实头像）；点击头像弹出菜单，内含退出。未登录时仅显示「登录」入口。**不再使用顶栏**，全局操作集中在左侧。

**Workspace** 是 Shell 右侧的全部剩余空间，不同页面在这里有完全不同的布局策略。Workspace 不是一个固定的模板，它根据当前任务的性质做出结构性的变化——这是整套布局方案最核心的设计决策：**布局本身就是页面功能的视觉隐喻**。

------

### 探索页（公开素材库）

这是平台唯一有 SEO 需求的核心页面，也是新用户的第一印象。

**布局策略：全宽沉浸式**

顶部没有传统的筛选栏，取而代之的是一个**全宽的横向标签滚动条**（像游戏分类标签），标签是素材的风格分类，用 DM Mono 小字渲染，当前选中的标签用青绿色下划线标记，不用背景色填充。

主体是砖墙网格（Masonry Grid），列数随视口宽度自适应，但最小列宽固定为 280px——保证卡片有足够空间展示素材名称的 Bebas Neue 大字。砖墙不设固定行高，完全由内容决定，有图和无图的卡片自然形成高低错落的节奏，像游戏展览的宣传墙。

网格与左侧 Shell 之间没有额外的侧边栏，探索页是整个平台最"开阔"的页面，给用户一种进入了一个广场的感觉。

搜索入口不是顶部的搜索框，而是键盘快捷键 `Cmd+K` 唤起的全局命令面板，保持内容区上沿的干净。

### 我的库页

**布局策略：侧边树 + 主内容区，二栏式**

这是平台里信息密度最高的页面，用户需要在多个维度管理自己的素材：按创建时间、按会话来源、按 fork 关系。

左侧增加一个**二级侧边栏**（200px 宽），它在 Shell 的右侧展开，和 Shell 一起构成平台左侧的"导航区域"（总宽 248px）。二级侧边栏里是树状结构：「全部素材」作为顶层节点，下方是按照 AI 会话来源分组的子节点（会话名称），以及一个「未归类」的兜底分组。树状节点用 12px DM Mono 渲染，左侧用 2px 的竖线表示层级，而不是传统的缩进箭头。

右侧主内容区是素材网格，和探索页不同，我的库的网格**有固定行高**——因为这里用户需要精确操作（批量选择、拖拽排序），固定高度的卡片比砖墙更可控。卡片尺寸统一为 240×280px，4 列。

主内容区顶部有一个轻量的工具栏：视图切换（网格 / 列表）、排序方式、以及一个"新建素材"的主操作按钮。这个按钮是全库页面唯一使用青绿色填充的元素，视觉权重最高。

### 素材详情页

**布局策略：左右分屏，比例动态**

这个页面最重要的设计决策是：**左右两栏的宽度比不是固定的**，用户可以拖动中间的分割线来调整。默认比例是 45%（文本区）/ 55%（图像区）。

左侧文本区的内部分区从上到下依次是：素材名称（Bebas Neue 大字，可点击进入编辑模式）、fork 来源信息（小字，有则显示）、描述区（DM Mono，多行文本，可编辑）、分隔线、注释区（折叠态，有注释时展开，颜色比描述区略暗，视觉上区分"不参与生图"的性质）。

左侧区域的高度几乎总是少于右侧，因为文本内容有限。剩余空间不填充任何内容，保持空白——空白本身是"文本优先"原则的视觉证明。

右侧图像区是横向滚动的图像列表，每张图等宽等高（正方形，约 260px）横向排列。列表下方是附加描述记录（折叠式，点击某张图展开查看它的生图参数）。列表最右侧始终有一个"+ 生图"的虚线框卡片，点击展开底部的生图输入面板（不是弹窗，是从底部推入的 drawer）。

页面顶部有一个轻量的面包屑导航：「我的库 / [会话名称] / 素材名称」，DM Mono 小字，用 `/` 分隔，点击任意层级可跳转。面包屑右侧是操作区：「公开」按钮（高风险，有视觉阻力）和「Fork」按钮。

### AI 创作会话页

**布局策略：主副分屏，左重右轻**

这是平台里交互最复杂的页面，空间分配需要反映工作流的优先级。

左侧聊天区占 **58%** 宽度，是主区域。右侧素材暂存区占 **42%**，是副区域。分割线是固定的，不可拖动——因为聊天区和暂存区的宽度需要经过视觉校准，确保双方都有足够的呼吸空间，不应该交给用户随意调整。

**左侧聊天区**分为两部分：上方的消息流区域（可滚动，占大部分高度）和底部的固定输入区。消息流区域没有顶部标题栏，只有极顶部一行小字显示会话主题名称（DM Mono，居中，颜色极浅）。AI 输出的素材卡片在消息流中渲染为内嵌的结构化卡片，宽度是消息气泡的 80%，左对齐。

底部输入区高度固定为 80px，包含文本输入框和发送按钮。输入框没有边框，只有底部一条 1px 的青绿色线条（像游戏 UI 里的输入框风格），获得焦点时这条线产生轻微的扫描动画。

**右侧素材暂存区**在会话开始时是空的，显示一句引导文字"与 AI 对话后，提炼的素材会出现在这里"。第一条素材出现时，暂存区从右侧滑入，整个右侧区域有一个 300ms 的展开动画。暂存区顶部有"全部导出"按钮，每条素材卡片右侧有独立的"导出"按钮。暂存区的卡片是精简版，只有名称和描述的前两行，不展示图像区域。

### 情绪板画布页

**布局策略：全屏画布 + 浮动面板系统**

这是唯一一个完全脱离全局框架的页面——连左侧的 Shell 导航栏都会收起（用 `Cmd+\` 可召回），把所有像素都让给画布。这个决策背后的逻辑是：用户进入画布意味着进入了"创作模式"，此时任何固定的 UI 元素都是干扰。

画布本身是深靛蓝紫色背景（`#080B14`）上的细点阵，点阵颜色是 `rgba(0, 255, 178, 0.06)`——极度克制的青绿色点，在黑暗中隐约发光，像某种科幻游戏的坐标系。

**浮动面板系统**由三个独立面板构成，每个面板都可以拖动到画布上的任意位置：

- **素材库面板**（默认吸附左侧）：可搜索的私人素材库，卡片可直接拖入画布。宽度 280px，高度占据视口 70%，顶部有搜索框。
- **工具栏**（默认吸附顶部居中）：选择、移动、文字标注、连线（不默认开启）、缩放等基础工具，水平排列，背景半透明深色，宽度随工具数量固定。
- **AI 协同面板**（默认收起，吸附右侧）：以图标按钮形式存在于画布右边缘，点击后展开为 320px 宽的 drawer，背景略深于画布，有明显的左边框（1px 紫罗兰色）区分 AI 区域。

三个面板之间没有层级关系，后点击的面板在 z 轴上置顶，完全平等的浮动系统。

画布左下角有一个**迷你地图**（120×80px），显示整个画布的素材分布全局视图，当前视口用一个白色矩形框标记位置，点击迷你地图上的区域可以跳转视角。这个功能对素材较多时的大型情绪板非常重要，同时也是对游戏开发者的一个小小的"致敬"——游戏里无处不在的小地图。

页面左上角有一个极小的退出按钮（`←`），点击后确认是否保存，然后返回项目总览页。这是画布页唯一的导航出口。

### 布局系统的统一规则

最后，给出整套布局的底层网格规范，让所有页面的间距有统一的来源：

- **基础间距单位：** 4px
- **组件内部间距：** 8px、12px、16px（2x、3x、4x）
- **组件之间间距：** 24px、32px（6x、8x）
- **区域之间间距：** 48px（12x）
- **最大内容宽度：** 无限制——平台是全宽设计，不设 max-width，因为大屏幕给了更多的画布空间，不应该浪费
- **断点：** 只有一个断点，1280px。低于此宽度时侧边栏折叠，二级侧边栏隐藏，网格列数减半
