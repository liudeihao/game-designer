## 前端完整开发方案

### 技术选型

**框架：Next.js 15 (App Router)**，配合 TypeScript。Next.js 的 RSC（React Server Components）非常适合公开素材库的 SEO 需求，同时客户端组件处理画布等重度交互。

**UI 基础：** 自建设计系统，不用 Shadcn/UI 那套过于通用的风格——用 **Radix UI Primitives**（无样式的无障碍组件原语）+ 自定义 CSS 变量，保证视觉完全可控。

**画布：** 阶段三的情绪板画布用 **tldraw**（开源的无限画布库），它本身就是为设计工具设计的，可以深度定制自定义卡片节点。

**状态管理：** Zustand（轻量）+ React Query / TanStack Query（服务端数据与缓存）。

**动画：** Framer Motion（页面转场、卡片交互）。

**图标：** Lucide React。

### 视觉设计语言（修订版）

##### 定位重新校准

参照系是：**游戏引擎的编辑器 + 独立游戏的官网 + 地下杂志的排版**。具体来说，是 Godot 编辑器的深色氛围、itch.io 个人页面的个性张扬、以及《极乐迪斯科》美术书的印刷质感的某种混合体。

目标感受：**打开这个平台的瞬间，用户应该感觉到"这是为我造的"，而不是"这是一个专业工具"。**

------

##### 色彩

色彩体系以**高饱和度的深色**为基础，而不是低饱和度的深灰；**主品牌色为冷蓝**，辅以**琥珀**与**紫罗兰**，避免单一青绿主导全站。

- **背景层：** **极深的靛蓝紫** `#080B14`，与实现一致
- **表面层（卡片、面板）：** `#0F1420`
- **品牌强调色（`--accent`）：** **冷蓝**（深色模式约 `#5B9DFF`，浅色模式约 `#2563EB`）——导航选中、主链接、主按钮、焦点环与分隔条 hover；取代原先大面积使用的霓虹青绿
- **次要冷色（`--sky`）：** 更浅的蓝，用于与 accent 同色相的层次（背景光晕、次要描边）
- **暖色（`--ember`）：** **琥珀**（深色模式偏金 `#F0BE5C`，浅色模式 `#D97706`）——计数、暂存提示、需与主蓝区分的温和强调；背景有微弱琥珀光晕（`--bg-glow-ember`）
- **AI / 光谱（`--ai-violet`）：** 紫罗兰，会话气泡、Fork 谱系等「AI / 分叉」语义保持不变
- **文本：** 主文本 `#E2E8F4`（深模式）/ 深青灰（浅模式）
- **边框（`--border`）：** 低透明度**蓝**相，与 accent 一致方向

核心逻辑仍是 **大面积克制、小面积高亮**，但高亮由**蓝 + 琥珀 + 紫**分工，而不是全站青绿。

**浅色模式：** 「我的库 → 显示设置」切换；`globals.css` 中 `html[data-gd-theme="light"]` 与深色共用变量名（`accent` / `sky` / `ember` / `ai-violet` 等）。`data-gd-theme`、`localStorage` 键 `gd-ui-prefs` 不变；**tldraw** 与 **Sonner** 仍跟随主题。

------

##### 字体

彻底更换字体选型，Instrument Serif 太文艺，Geist 太科技公司。

- **展示字体（大标题、素材名称）：** **Bebas Neue**——全大写的冷凝无衬线体，在游戏圈极度熟悉（大量独立游戏标题使用类似字体），有力量感而不失现代感。素材名称用它渲染，在砖墙布局里有强烈的视觉节奏
- **正文 UI 字体：** **DM Mono**——等宽字体用在 UI 里，而不是传统的无衬线体。这个选择有点反常规，但非常"游戏开发者"——他们整天看代码，等宽字体对他们来说反而有亲切感，而且等宽字体的规整节奏配合游戏工具的语境非常协调
- **描述性长文本：** **Inter**（这里反而用回常规选择）——因为素材描述是需要认真阅读的内容，等宽字体读长文本会疲劳，Inter 的可读性在这个场景是优势
- **AI 输出内容：** 保持 DM Mono，强化"这是机器输出"的感知

字体的整体效果：Bebas Neue 的大标题 + DM Mono 的 UI 文本，会让界面有一种**"独立游戏 README + 游戏编辑器"**的独特混搭气质。

------

##### 材质与纹理

独立游戏开发者对纹理有本能的好感——像素噪点、扫描线、胶片颗粒感，这些在他们的游戏里司空见惯。

- **全局噪点叠层：** 在 `body` 上用 CSS 伪元素叠加一层极低透明度（约 3%）的 SVG 噪点纹理，让整个界面有轻微的颗粒感，消除数字屏幕的过于光滑感
- **素材卡片的无图占位块：** 不再是简单的渐变色，改为**程序化生成的像素噪点图案**——用 Canvas API 根据素材名称的哈希值生成不同的彩色噪点种子，像早期游戏的随机地图生成一样。每个无图素材都有独一无二的"像素指纹"
- **面板边缘：** 关键面板（如 AI 会话窗口的边框）用细线 + 轻微的扫描线效果（`repeating-linear-gradient` 实现），不过度，只是一丝存在感

------

##### 动效语言

需要**更有个性，但不浮夸**。

- **页面切换：** 不用普通的淡入淡出，改用**水平扫入**——新页面从右侧快速扫入（约 200ms，有轻微的运动模糊感），像游戏的场景切换
- **卡片 hover：** 边框从暗淡状态**瞬间提亮**（`border-accent/35` 等，随 `--accent` 为蓝），同时卡片整体上移 2px。不是渐变，是瞬间切换——像游戏里选中一个物件时的即时反馈，而不是过渡平滑的 UI 动画
- **按钮点击：** 品牌强调色的按钮点击时做一个极短的**闪白帧**（16ms 的白色叠加），模拟游戏里的"命中闪光"效果
- **AI 生成素材时：** 素材卡片从外框向内"扫描式"渲染，像老式打印机或游戏加载界面，而不是淡入

### 核心页面方案

#### 1. 公开素材库 & 我的素材库

**布局：** 左侧是持久化的工作区侧边栏（160px），右侧主内容区。侧边栏分层：公开库 / 我的库 / 会话 / 归档。

**素材卡片：** 这是全平台最核心的 UI 组件，两种形态：

+ **有图模式：** 图像占满卡片上 2/3，下方露出名称 + 简短描述的 overlay
+ **无图模式（关键）：** 根据调研报告建议，使用**程序化占位块**——根据描述文本的语义色（可以是前端简单的字符串哈希映射到色相值），生成带微妙噪点纹理的渐变色块，上方叠加素材名称。这样无图卡片在情绪板中依然有视觉重量，而不是白板上一个空文本框

**网格：** 响应式砖墙布局（masonry），用 CSS Grid + `grid-template-rows: masonry`（渐进增强，降级用 JS 方案）

**Fork 树入口：** 每张卡片 hover 时右上角出现小图标，点击展开一个 popover 展示该素材的上下游 fork 关系，用 minimap 形式呈现

#### 2. 素材详情页

**布局：** 左侧是素材的核心信息（名称、描述、注释区，可编辑），右侧是图像管理区。两栏并排，左窄右宽（40/60 分割）。

**图像管理区：** 横向滚动的图像列表（类似 Figma 的帧列表），每张图可展开查看生图时的附加描述记录。底部有一个"+ 生图"的操作区，输入附加描述 → 调用后端 API → 流式显示生图进度。

**AI 注释区：** 和描述区视觉上有所区分（用斜线分隔线或不同的背景），加上一个小图标提示"注释不参与生图"

#### 3. AI 创作会话

**布局：** 全屏的聊天+工作台混合视图。左侧是聊天区（60%），右侧是本次会话产生的素材暂存区（40%）——类似 Cursor 的编辑器+聊天分屏体验。

**素材暂存列表：** 右侧显示 AI 已提炼出的素材卡片（只有名称和描述），每张卡片有"导出到我的库"的按钮，也可以全选批量导出。

**聊天气泡：** AI 生成素材时，在气泡中内嵌一个结构化的素材卡片预览，而不是纯文本——让用户直观看到"这就是一条素材"。

#### 4. 情绪板画布（阶段三核心）

**技术：** 基于 tldraw，注册自定义的"素材节点"类型。从私人库拖拽素材到画布时，渲染为自定义卡片（有图则显示图，无图则显示程序化色块），并保持和素材库的数据绑定。

**操作流：** 右侧浮动面板可以搜索/浏览私人素材库，直接拖入画布。AI 协同对话以 slide-over drawer 形式出现，不遮挡画布主体。

**视觉：** 画布背景用细点阵（dot grid），不用线条网格——更接近设计工具的手感。

### 页面路由结构

```
/                          # 落地页 / 用户未登录
/explore                   # 公开素材库（探索）
/library                   # 我的库（重定向到 /library/assets）
/library/assets            # 我的素材
/library/assets/[id]       # 素材详情（阶段一+二）
/library/sessions          # AI 会话列表
/library/sessions/[id]     # 单个 AI 会话工作台
/library/sessions/archive  # 归档会话
/projects                  # 项目列表（阶段三）
/projects/[id]/canvas      # 情绪板画布（全屏）
/projects/[id]/settings    # 项目设置
/u/[username]              # 用户主页 / 公开 profile
```

### 关键工程决策

**阶段一就要实现的核心组件：** `AssetCard`（两种模式）、`ForkBadge`（显示 fork 来源）、`ProcedualPlaceholder`（无图占位色块）、`AssetDetailPanel`、`ChatSession`。

**API 通信：** 标准 REST（Go 后端），图像生成进度用 SSE（Server-Sent Events）做流式更新，避免 WebSocket 的复杂度。

**国际化：** 先做中文，用 next-intl 预留 i18n 结构。

**无障碍：** 所有交互元素有 keyboard navigation，颜色对比度满足 WCAG AA（深色主题下相对容易做到）。

### 分阶段交付建议

**第一批（配合阶段一后端）：** 设计系统基础（颜色、字体、组件库文档）→ 布局框架（侧边栏+主区域）→ AssetCard 组件 → 公开库 / 我的库页面 → 素材详情页（纯文本部分）→ AI 会话页面。

**第二批（配合阶段二后端）：** 素材详情页图像管理区 → 生图工作台 → 图像版本历史。

**第三批（配合阶段三后端）：** 项目页面 → 画布（tldraw 集成）→ 素材导入面板 → AI 协同 drawer。
