没有组件规范，不同开发者会对同一个 UI 元素做出不同的实现决策，最终产品会在细节上四分五裂。

组件按照使用频率和复杂度分层来规划。

------

### 第一层：原子组件

这些是最小的、不可再拆分的 UI 单元，所有复合组件都由它们组成。

#### Tag / 标签

平台里有三种语义的标签：风格分类标签（在探索页）、状态标签（公开 / 私有 / 已删除）、来源标签（fork 来源提示）。三种标签共享相同的尺寸规格，但颜色语义不同。

规格：高度 20px，水平内边距 8px，字号 11px DM Mono，圆角 3px（不是胶囊形，保持硬朗）。

颜色语义：风格分类标签用极低透明度的青绿色背景（`rgba(0,255,178,0.08)`）+ 青绿色文字；状态"公开"用同样逻辑的紫罗兰色；状态"私有"用中性灰；状态"已删除"用划掉的文字样式 + 更深的灰，不用红色（红色在游戏语境里是"危险 / 血量"，用在"已删除"会造成认知偏差）。

#### IconButton / 图标按钮

平台里大量操作以图标按钮形式存在（复制、fork、导出、生图等），需要一套统一的规格。

规格：32×32px 的点击热区，图标本身 16px。背景透明，hover 时出现 `rgba(255,255,255,0.06)` 的圆形背景（不是方形，圆形在深色背景上更柔和）。激活状态（如当前工具被选中）背景用 `rgba(0,255,178,0.12)`，图标颜色切换为青绿色。

不要有边框——图标按钮的边框在深色背景上非常廉价，透明背景 + hover 色变化足够清晰。

#### Divider / 分隔线

只有一种分隔线：1px 高度，颜色 `rgba(255,255,255,0.06)`，无需任何变体。禁止使用不同粗细或不同颜色的分隔线——整个平台应该只有这一种分隔线，用空间距离来区分层级，而不是用不同的线。

#### Select / 下拉选择

**工作区内禁止原生 `<select>` / `<option>`**（浏览器默认样式与主题不一致）。统一使用 `web/src/components/ui/ThemeSelect.tsx`（Radix Dropdown + 主题边框/背景）；工具条等紧凑处通过 `className` 覆盖 `text-xs`、`py-1.5` 等即可。

#### StatusDot / 状态点

用于表示异步操作的进行状态（图像生成中、AI 处理中、已保存），比转圈 Loading 更轻量。

规格：6px 直径的圆点。三种状态：脉冲动画（处理中，青绿色）、静止（完成，青绿色）、静止（失败，用不饱和的暗红色，区别于游戏语境里的鲜红）。圆点始终配合一个 DM Mono 11px 的状态文字，不单独出现。

### 第二层：核心业务组件

#### AssetCard / 素材卡片

这是整个平台最重要的组件，出现频率最高，设计决策影响最大。它有三种形态，共享同一套基础结构。

**基础结构：** 卡片是一个矩形容器，圆角 6px，背景 `#0F1420`，边框 `rgba(0,255,178,0.10)`。没有 box-shadow，用边框代替阴影——阴影在深色背景上几乎不可见，而且会让界面变重。

**形态一：有图卡片（GridCard）** 用于探索页和我的库的网格视图。

图像区占卡片上方约 65% 高度，图像以 `object-fit: cover` 填充，不留白边。图像和文字区之间没有分隔线，用轻微的渐变遮罩过渡（从透明到 `#0F1420`，高度 40px）。

文字区在图像下方，内边距 12px。素材名称用 Bebas Neue，字号 18px，颜色 `#E2E8F4`，最多显示一行，超出截断。名称下方是描述的前两行，Inter 12px，颜色 `rgba(226,232,244,0.5)`，第二行末尾用渐隐截断而不是省略号。

卡片底部栏（高度 28px，内边距 12px）显示作者头像（16px 圆形）+ 作者名（DM Mono 11px）以及 fork 来源标签（如果有）。

Hover 状态：边框透明度从 0.10 提升到 0.35，卡片整体上移 2px，这两个变化同时发生、无过渡——是瞬间切换，不是动画。底部栏同时出现快捷操作图标：Fork、收藏、查看 Fork 树，从右侧淡入。

**形态二：无图卡片（ProceduralCard）** 无图时不显示空白区域，而是渲染程序化生成的像素噪点图案。

实现方式：用素材 ID 作为随机种子，通过 Canvas 绘制抽象几何与波形（`ProceduralPlaceholder.tsx`）。色相取自预设盘，饱和度约 40% 以内。`html[data-gd-theme="light"]` 时使用高明度背景（约 90–95% L）与较深描边/叠色；深色模式保持低明度底；`MutationObserver` 监听主题属性以便切换配色时重绘。

这个像素图案是该素材的"视觉指纹"，每个素材独一无二，用户在浏览多次后会开始对这个图案产生记忆——这是有意设计的，即使没有图也应该让素材具备可识别性。

文字区和底部栏与有图卡片完全相同，不做区分，因为无图不是"残缺"状态，只是另一种形态。

**形态三：精简卡片（CompactCard）** 用于 AI 会话暂存区和素材导入面板。

整体高度固定 64px，横向布局。左侧是 48×48px 的正方形（有图则显示缩略图，无图则显示 24×24px 的像素噪点图案，不是填满的）。右侧是名称（Bebas Neue 14px）+ 描述第一行（Inter 11px，颜色 50% 透明）。最右侧是操作图标。

这三种形态共享同一个 React 组件，通过 `variant` prop 切换，不是三个独立组件。共享部分（hover 逻辑、点击行为、数据结构）不重复实现。

------

#### AssetGrid / 砖墙网格

网格容器组件，负责砖墙布局的实现和无限滚动。

砖墙布局用纯 CSS 的 `columns` 属性实现，而不是 JS 计算。`columns: 4` 在宽视口下，`columns: 2` 在窄视口下（低于 1280px）。这个方案的优势是不需要 JS 计算每张卡片的位置，性能更好，但列顺序是竖向的（第一列排满再排第二列），在语义上对用户没有影响。

列间距 16px，卡片之间的垂直间距也是 16px（用 `margin-bottom` 实现）。

无限滚动通过 IntersectionObserver 监听列表末尾的一个哨兵元素，进入视口时触发 TanStack Query 的 `fetchNextPage`。加载下一页时，新卡片从底部淡入（opacity 0→1，duration 200ms，stagger 20ms 每张），不跳动，不重排已有卡片。

空状态（没有任何素材时）：不用插图，用居中的一段文字 + 一个操作按钮。文字是 DM Mono，内容根据当前上下文变化——探索页空状态说"暂无公开素材"，我的库空状态说"写下第一个概念的名称"。

------

#### ForkBadge / Fork 关系徽章

显示素材的 fork 来源，出现在素材卡片底部和素材详情页顶部。

当素材有 `forkedFromId` 时显示。样式是一个小箭头图标（`↑` 样式，表示来自上游）+ DM Mono 11px 的来源素材名称，整体可点击，点击跳转到来源素材详情页。

如果来源素材已被删除（`visibility: 'deleted'`），显示为"衍生自 [已删除的素材]"，文字加删除线样式，不可点击，颜色更暗。这条信息依然显示，不因为来源被删除而消失——这是调研报告里强调的"创意溯源的尊重"。

------

#### ForkTree / Fork 关系树

在素材详情页可展开查看，或在独立的弹出层里全屏查看。

组件内部用 SVG 绘制树状关系图。当前素材节点居中，上方显示直接上游（最多一层，因为一个素材只有一个 forkedFromId），下方显示直接下游（可以有多个，横向排列）。节点样式和 AssetCard 的精简形态一致（48px 高的横条），用青绿色细线连接。

如果下游节点超过 4 个，超出部分折叠为"还有 N 个衍生"的点击节点，点击后展开并重新布局，用动画过渡新节点的出现。

当某个节点是已删除素材时，节点渲染为虚线边框 + 灰色文字，点击无效果，但连接线依然存在——树的结构不因节点删除而断裂。

------

#### ChatMessage / 聊天消息

AI 会话页面的消息气泡组件，有两种子类型。

**用户消息：** 右对齐，最大宽度 70%，背景 `rgba(255,255,255,0.04)`，内边距 12px 16px，圆角 6px（右下角圆角减为 2px，模拟传统聊天气泡的"尾巴"方向感）。字体 Inter 14px。

**AI 消息：** 左对齐，无背景（直接在聊天区背景上），无气泡边框。字体 DM Mono 14px，行高 1.8（等宽字体需要更大行高才舒适）。纯文本内容以打字机效果逐字渲染。

当 AI 消息中包含素材时，素材卡片以 CompactCard 形态内嵌在消息流里，宽度 80%，左对齐，上下各有 8px 的外边距与文字内容分隔。素材卡片出现时有一个从左侧扫入的动画（translateX -12px → 0，duration 150ms），区别于文字的打字机效果，明确告知用户"这是一个结构化对象，不是文字"。

消息流的时间戳只在 hover 时才在每条消息右侧淡出显示，不常驻——常驻时间戳会制造信息噪音，但完全不显示又可能让用户找不到某条消息的时间参考。

------

#### ImageStrip / 图像横向列表

素材详情页右侧的图像管理组件。

横向滚动容器，高度固定 280px（图像正方形 240px + 下方信息栏 40px）。每张图之间间距 12px，左侧无内边距（图像顶到容器边缘，产生"胶卷"感）。

每张图在 hover 时出现两个操作图标：设为封面（星形图标）和查看详情（展开图标）。点击查看详情时，图像以 lightbox 形式在画布中央展开，同时右侧滑入该图的生图参数记录（附加 prompt、生成时间）。

最右侧的"+ 生图"卡片是虚线边框，宽度与其他图片相同（240px），点击后画布底部推入一个生图输入面板（不是弹窗，是从底部向上推入 160px 的 drawer），包含一个输入框和发送按钮。生图请求发出后，列表最右侧（在"+ 生图"按钮左边）出现一个新的"生成中"占位卡，用 StatusDot 的脉冲动画表示进度，完成后图像以淡入效果替换占位卡。

------

#### CanvasAssetNode / 画布素材节点

tldraw 的自定义节点，是情绪板画布里的核心元素。

节点整体是一个圆角矩形（圆角 6px），边框默认 `rgba(0,255,178,0.15)`，选中时边框变为实青绿色并加粗到 1.5px。节点的默认尺寸是 200×240px，可以被用户自由缩放。

节点内部分两区：上方 170px 是图像区（有图显示图像，无图显示像素噪点图案，实现和 ProceduralCard 一致），下方 70px 是信息区，背景略深（`#080B14`），内边距 10px，显示素材名称（Bebas Neue 14px）和描述第一行（Inter 11px，50% 透明）。

当画布缩放比例低于 40% 时，节点切换到"色块模式"——只显示图像区或像素图案，隐藏信息区，节点高度缩为 160px。这个切换有 100ms 的淡入淡出过渡。缩放高于 100% 时，节点展开显示完整描述（展开信息区到 120px，显示描述的前三行）。这三种缩放层级的信息密度设计，让情绪板在任何视距下都有意义。

节点右键菜单只有三项：「在库中查看」、「从画布移除」、「复制 ID」。不提供"编辑"——编辑只能在素材库里进行，维持数据来源的唯一性。

------

#### CommandPalette / 命令面板

全局 `Cmd+K` 唤起的搜索与导航组件，是平台最重要的效率工具。

出现方式：页面中央的模态层，背景有 `rgba(0,0,0,0.6)` 的遮罩，面板本身 560px 宽，顶部对齐视口 20% 高度处（不居中，略偏上，让用户视线自然落在输入框上）。

面板内部：顶部是搜索输入框（占满面板宽度，高度 48px，无边框，DM Mono 16px，光标用青绿色），下方是结果列表（每项高度 44px，最多显示 8 项，超出可滚动）。

结果列表支持四种类型的结果项：素材（显示像素图案或缩略图 + 名称）、会话（显示会话主题名称）、项目（显示项目名称）、操作（如"新建素材"、"新建会话"，这些操作项始终出现在结果列表底部，不受搜索词影响）。

当搜索框为空时，默认显示"最近访问"列表（最多 5 项），让命令面板也承担快速跳转的职责，而不只是搜索工具。

键盘导航：方向键移动焦点，Enter 执行，Escape 关闭。任何时候按 Escape 都能关闭，不需要点击遮罩。

### 第三层：页面级复合组件

这一层不做详细拆解（因为它们是第一层和第二层的组装），只列出需要作为独立组件管理的页面区块：

- **WorkspaceSidebar**：全局左侧导航栏（Shell 层的 48px 图标栏）
- **LibrarySidebar**：我的库的二级侧边栏（200px 树状导航）
- **SessionWorkspace**：AI 会话页面的整体布局（包含聊天区 + 暂存区的分屏逻辑）
- **CanvasFloatingPanels**：画布页面的三个浮动面板的管理容器（负责 z 轴管理和位置持久化）
- **MiniMap**：画布左下角的全局视图，独立组件便于独立测试

### 组件文档的约定

最后一点：每个组件在代码库里必须对应一个 Storybook story，story 覆盖所有 variant 和状态（loading、error、empty、hover、active、disabled）。这不是为了文档而文档，而是因为平台的深色背景在浏览器里和在 Storybook 里的渲染效果可能有差异，所有视觉状态都必须在隔离环境里验证过，才能认为组件"完成"。
