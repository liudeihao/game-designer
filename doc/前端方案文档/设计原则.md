### 一、信息层级的核心哲学：文本优先，图像附属

调研报告里有一个洞察非常准确：**图像地位降级会造成"视觉失语症"**。这对 UI 设计的启示是：不能让无图素材在视觉上显得"残缺"，但也不能为了填补空白而让图像反客为主。

解决这个矛盾的原则是：**所有素材卡片的主体永远是名称**，图像只是名称的"视觉注脚"。具体体现为：

+ 有图卡片里，图像是背景，名称永远在前景、永远可读
+ 无图卡片的占位色块，颜色饱和度刻意压低（不超过 30% 饱和度），确保它提供视觉重量的同时，不会喧宾夺主地被误认为"内容图"
+ 素材详情页的布局，名称和描述永远在左列，图像在右列——即使图像很精美，视线的起点也是文字

这个原则还要渗透到空状态设计里：新用户还没有任何素材时，空状态文案不该是"上传图片开始"，而应该是"写下第一个概念的名称"。

------

### 二、渐进式披露：不要一次暴露所有复杂度

平台有一个隐藏的复杂度陷阱——fork 机制、版本树、会话归档、图像版本历史——这些概念对新用户来说信息量很大，但对熟练用户又是核心工作流。

原则是：**默认界面只展示"当下任务所需的最少信息"，复杂度藏在第二层**。

几个具体的体现：

+ 素材卡片默认不显示 fork 来源信息。只有用户主动 hover 或点击"查看关系"时，才展示 fork 树。这叫做**延迟披露**
+ 素材详情页的"注释"区域默认折叠，有注释时才展开，避免让用户觉得注释是必填项
+ AI 会话页面的素材暂存区，产生第一条素材之前是收起状态，产生后才滑入出现——功能在需要时出现，不需要时不占位
+ Fork 树可视化（复杂的图结构）只在用户明确想"溯源"时才打开，不把它放在主流程里

------

### 三、状态可见性：每个对象都该知道自己"是什么"

平台里的素材存在多种状态：私有草稿 / 已公开 / 来自 fork / 被 fork 过 / 有图 / 无图 / 图像生成中。这些状态如果用 badge 堆砌，会造成视觉噪音。

原则是：**在正确的场景只显示当前最重要的一个状态信号**，其余状态在详情页或 tooltip 里补充。

状态信号的视觉语言也要统一：

+ **来源信息**（fork 自哪里）用细体小字 + 链接色，放在卡片底部——不重要但可追溯
+ **可见性**（公开 / 私有）只在"我的库"语境下显示，公开库里所有素材都是公开的，不需要标注
+ **图像生成中**的状态用微弱的脉冲动画（不是转圈 loading），因为生图是异步的、可能需要几秒到几十秒，转圈会让用户感到焦虑

------

### 四、操作的可逆性设计

平台有一些"高风险"操作：删除已公开的素材（会造成下游 fork 的幽灵节点）、公开素材（不可撤回变为不可修改）。

原则是：**风险越高的操作，阻力越大，但不要用弹窗轰炸**。

具体实现逻辑：

+ **公开素材**：这是不可逆的重要动作。按钮本身要有足够的视觉分量（不是文字链），点击后进入一个确认态——用内联展开的方式（在原按钮下方展开确认区），而不是打断注意力的模态弹窗
+ **删除公开素材**：需要用户手动输入素材名称才能确认——借鉴 GitHub 删除仓库的交互，因为这个操作对社区有影响
+ **删除私有素材**：普通 undo toast，5 秒内可撤销，不需要确认弹窗
+ **生成图像**：明确展示"这会消耗一次生图额度"，让用户对资源消耗有预期，而不是在用完后才发现

------

### 五、AI 会话的角色定位：工具，不是聊天机器人

AI 会话页面的设计很容易滑向"做一个好看的 ChatGPT 界面"，但这是错误方向。这里的 AI 是**专门服务于素材提炼的工具**，不是通用对话伙伴。

设计原则体现在：

+ **输入框的 placeholder** 不应该是"输入消息…"，而是"描述你想要的素材，例如：赛博朋克风格的废弃加油站"——给用户方向感
+ **AI 回复的呈现格式**：当 AI 提炼出素材时，它不该只输出纯文本，而是在消息流中直接渲染一个内嵌的素材卡片预览（名称+描述的结构化展示），让用户立刻能识别"这是一条素材成果"
+ **会话历史不是主体**：聊天气泡区的视觉权重应该低于右侧的素材暂存区，因为对话本身是过程，素材才是产物。可以用较小的字号、较浅的颜色来实现这种权重差异
+ **主题感**：会话可以有主题名称（例如"废土风格道具系列"），这个主题名称作为页面标题大字展示，强化用户的创作焦点感

------

### 六、情绪板画布的设计原则：空间就是语言

画布是平台最自由的区域，但"自由"不等于"无规则"。

+ **缩放层级要影响信息密度**：缩小到 30% 以下时，卡片自动简化为只显示色块（占位色或图像缩略图），不显示文字——这是情绪板的"宏观视角"，看整体色彩构成；放大到 100% 以上时，完整展示名称和描述
+ **素材卡片在画布上是只读的**：不能在画布里编辑素材内容。想编辑必须跳回素材库。这条原则防止"在哪里改了什么"的混乱，数据来源永远是素材库
+ **连线功能要克制**：不要默认提供节点连线功能，因为这容易让画布退化成流程图工具。如果要表达关联，用空间位置来隐喻（把相关素材放在一起），而不是显式的箭头
+ **AI 侧边栏的可见性**：AI 协同对话默认收起，不占据画布空间。用户召唤它时以 overlay drawer 形式出现，对话结束后自动收起——画布是主角，AI 是助手

------

### 七、社区生态的设计原则：贡献感而非竞争感

公开素材库不是一个排行榜，不应该强化"哪个素材最受欢迎"的竞争气氛，而应该强化**"这条素材从哪里来、去了哪里"**的文化传承感。

+ **不显示绝对数字**（"被 fork 了 238 次"），只显示相对关系（"有活跃的 fork 衍生"）——避免马太效应，不让热门素材越来越热、冷门素材完全被忽视
+ **Fork 来源永远可见**：即使原始素材被删除，卡片上也保留一行"衍生自 [已删除的素材]"，用删除线或 ghost 样式表达——这是对创意溯源的尊重
+ **用户 Profile 页面**的核心展示不是"他有多少素材"，而是"他的素材衍生出了哪些有趣的变体"——展示影响力，而不是数量

------

### 八、一致性底线：必须统一的 5 个细节

最后，无论哪个页面，以下 5 点必须绝对一致，否则平台会显得"拼凑感"：

1. **素材名称永远用 Bebas Neue 展示体（`font-display`，全大写节奏）渲染**，无论出现在卡片、详情页还是画布节点——这是整个平台最重要的视觉锚点，代表"概念的重量"（与《方案选型》《组件规范》一致，替代早期文档中的 Instrument Serif 表述）
2. **所有异步操作的反馈延迟不超过 100ms**——按钮点击后立即有响应（即使数据还没回来），用 optimistic update 策略
3. **空状态不用插图**——只用文字和简洁的几何符号，保持整体的克制感；插图风格很难统一，且容易让工具显得"不够专业"
4. **错误信息永远说"发生了什么"和"用户能做什么"**，而不是只说错误代码
5. **深色模式是唯一模式**（至少 v1 阶段）——不提供浅色切换，因为两套主题会成倍增加设计和开发成本，而目标用户（设计师）使用深色工具的比例极高
